# 卷详细大纲模板

本模板用于 `outlines/vol-{N}.md` 文件，融合**麦基戏剧理论**与**网文技法**，提供单卷的详细章节规划。

---

## 文件格式

```markdown
# 第{N}卷: {卷名} 详细大纲

> 所属作品: 《{书名}》
> 章节范围: 第{起始}-{结束}章
> ��成时间: {YYYY-MM-DD}

---

| 属性 | 描述 |
|------|------|
| 章节数 | {X}章 |
| 目标字数 | 约{X×3000}万字 |
| 核心冲突 | {本卷主要矛盾} |
| 主角境界 | {起始境界} → {结束境界} |
| 主要场景 | {场景列表} |

{200字左右的本卷完整剧情概述}

## 详细章节

### 第{起始}章: {章节标题}

```yaml
标题: {章节标题}

主要内容:
  - 开篇: {场景设定，100字}
  - 冲突: {本章核心冲突}
  - 发展: {情节推进}
  - 结尾: {章末状态}

出场角色:
  - {角色1}: {本章作用}
  - {角色2}: {本章作用}

# ===== 戏剧结构（麦基理论）=====

戏剧结构:
  场景极性: "{正/负}→{正/负}"  # 必填，如: 负→正
  开始价值: "{开始时的价值状态}"
  结束价值: "{结束时的价值状态}"

节拍设计:  # 重要场景填写，普通场景可省略
  - 行动: "{角色行动}"
    反应: "{世界/他人反应}"
    Gap: "{期望与结果的落差，制造惊奇}"
  - 行动: "{后续行动}"
    反应: "{反应}"
    Gap: "{落差}"

抉择设计:  # 有重要抉择的章节填写
  困境: "{两难困境描述}"
  选项A: "{选项A及其后果}"
  选项B: "{选项B及其后果}"
  主角选择: "{实际选择}"
  揭示: "{这个选择揭示了主角什么真实性格}"

# ===== 网文技法 =====

爽点:
  类型: {无/压抑铺垫/实力展示/获得机缘/打脸/突破}
  描述: {爽点具体内容}
  Gap分析: {期望落差如何制造爽感}  # 可选

章末钩子:
  类型: {悬念型/冲突型/预告型/反转型}
  内容: "{引导读者继续阅读的悬念句}"

境界: {主角当前境界}
场景: {主要场景}
字数: 3000字

与前后章关联:
  承接: {承接上一章什么内容}
  引出: {为下一章铺垫什么}
```

> **场景极性说明**:
> - `正→负`: 从好到坏（制造紧张）
> - `负→正`: 从坏到好（制造爽感）
> - `正→负→正`: 过山车体验
> - `负→正→负`: 制造绝望

## 本卷检查清单

### 戏剧结构检查
- [ ] 每章标注场景极性（正→负或负→正）
- [ ] 重要场景有节拍设计（行动→反应→Gap）
- [ ] 关键章节有抉择设计
- [ ] 价值转变与爽点对应

### 网文技法检查
- [ ] 每章有完整的主要内容描述
- [ ] 每章标注爽点类型和钩子
- [ ] 出场角色与characters.md一致
- [ ] 境界变化与outline.md一致
- [ ] 高潮节点数量达标（2-3个）
- [ ] 章节之间有关联承接
- [ ] 伏笔有埋有收
```

---

## 使用说明

1. 用户说"细化第X卷"时，读取 `outline.md` 中该卷的信息，生成完整章节大纲
2. 每章必填：标题、主要内容、出场角色、爽点、钩子、境界、场景、字数、前后关联
3. 生成后更新 `outline.md` 中的"待细化卷"状态
4. chapter-writer 创作时读取本文件获取章节详细规划
