# 卷详细大纲模板

本模板用于 `outlines/vol-{N}.md` 文件，融合**麦基戏剧理论**与**网文技法**，提供单卷的详细章节规划。

---

## 文件格式

```markdown
# 第{N}卷: {卷名} 详细大纲

> 所属作品: 《{书名}》
> 章节范围: 第{起始}-{结束}章
> 生成时间: {YYYY-MM-DD}

---

| 属性 | 描述 |
|------|------|
| 章节数 | {X}章 |
| 目标字数 | 约{X×3000}万字 |
| 核心冲突 | {本卷主要矛盾} |
| 主角境界 | {起始境界} → {结束境界} |
| 主要场景 | {场景列表} |

### 本卷剧情线

{200字左右的本卷完整剧情概述}

### 本卷高潮

3. ...

## 本卷结构设计

> 参考: `libraries/knowledge/_base/structure-principles.md`

### 五部分结构映射

| 结构元素 | 对应章节 | 描述 |
|----------|----------|------|
| 激励事件 | 第{X}章 | {打破本卷常态的事件} |
| 第一转折点 | 第{X}章 | {主角承诺追求本卷目标} |
| 中点 | 第{X}章 | {假胜利/假失败，改变游戏规则} |
| 第二转折点 | 第{X}章 | {最低点，获得关键洞察} |
| 高潮 | 第{X}章 | {本卷主要冲突的解决} |

### 复杂化模式

本卷采用: {障碍升级/揭示深化/内外交困}

```
复杂化链:
{简述各个复杂化节点，体现递进升级}
```

### 间隙类型分布

> 参考: `libraries/knowledge/_base/gap-theory.md`

| 章节 | 间隙类型 | 间隙描述 | 强度 |
|------|----------|----------|------|
| 第{X}章 | {反转/渐进恶化/延迟变形/同时性悖论/意义反讽/认知重构/规模错位/循环无解} | {简述} | ★★★☆☆ |
| 第{Y}章 | {类型} | {简述} | ★★★★☆ |
| ... | ... | ... | ... |

**间隙类型说明**:
- `反转`: 期待A得到相反的B
- `渐进恶化`: 不是反转而是持续加重
- `延迟变形`: 目标达成但方式/意义变质
- `同时性悖论`: A和B同时为真的撕裂
- `意义反讽`: 形式成功实质失败，或相反
- `认知重构`: 事实不变但理解彻底改变
- `规模错位`: 期待的规模与实际不匹配
- `循环无解`: 回到原点或根本无解

**间隙多样性检查**: 本卷使用了 {N} 种间隙类型（建议≥3种）

## 详细章节

### 第{起始}章: {章节标题}

```yaml
标题: {章节标题}

主要内容:
  - 开篇: {场景设定，100字}
  - 冲突: {本章核心冲突}
  - 发展: {情节推进}
  - 结尾: {章末状态}

出场角色:
  - {角色1}: {本章作用}
  - {角色2}: {本章作用}

# ===== 新实体规划（策划锁定，创作者必须遵守）=====
# 说明：明确规划本章出现的新实体，避免创作者随意发挥导致前后冲突

新实体:
  新角色:  # 本章首次出场的角色（蓝图characters.md中有但首次出场）
    - 名称: {角色名}
      身份: {角色身份}
      作用: {在本章的作用}
  新地点:  # 本章首次出现的地点
    - 名称: {地点名}
      作用: {在本章的作用}
  新物品:  # 本章首次出现的物品/法宝/丹药等
    - 名称: {物品名}
      作用: {在本章的作用}
  新技能:  # 本章首次出现的技能/功法
    - 名称: {技能名}
      持有者: {谁使用}
  # 如果本章无新实体，写 新角色: [] 即可

# ===== 戏剧结构（麦基理论）=====

戏剧结构:
  场景极性: "{正/负}→{正/负}"  # 必填，如: 负→正
  开始价值: "{开始时的价值状态}"
  结束价值: "{结束时的价值状态}"

节拍设计:  # 重要场景填写，普通场景可省略
  - 行动: "{角色行动}"
    反应: "{世界/他人反应}"
    Gap: "{期望与结果的落差，制造惊奇}"
  - 行动: "{后续行动}"
    反应: "{反应}"
    Gap: "{落差}"

抉择设计:  # 有重要抉择的章节填写
  困境: "{两难困境描述}"
  选项A: "{选项A及其后果}"
  选项B: "{选项B及其后果}"
  主角选择: "{实际选择}"
  揭示: "{这个选择揭示了主角什么真实性格}"

间隙设计:  # 本章的主要间隙（参考 gap-theory.md）
  类型: "{反转/渐进恶化/延迟变形/同时性悖论/意义反讽/认知重构/规模错位/循环无解/无}"
  期待: "{角色/读者的期待是什么}"
  实际: "{实际结果是什么}"
  落差: "{期待与实际的落差如何产生情感冲击}"
  # 如果本章无明显间隙，类型写"无"，其他字段省略

# ===== 网文技法 =====

爽点:
  类型: {无/压抑铺垫/实力展示/获得机缘/打脸/突破}
  描述: {爽点具体内容}
  Gap分析: {期望落差如何制造爽感}  # 可选

章末钩子:
  类型: {悬念型/冲突型/预告型/反转型}
  内容: "{引导读者继续阅读的悬念句}"

境界: {主角当前境界}
场景: {主要场景}
字数: 3000字

与前后章关联:
  承接: {承接上一章什么内容}
  引出: {为下一章铺垫什么}
```

> **场景极性说明**:
> - `正→负`: 从好到坏（制造紧张）
> - `负→正`: 从坏到好（制造爽感）
> - `正→负→正`: 过山车体验
> - `负→正→负`: 制造绝望

## 本卷检查清单

### 结构设计检查（参考 structure-principles.md）
- [ ] 五部分结构映射完整（激励事件→第一转折点→中点→第二转折点→高潮）
- [ ] 激励事件是否迫使主角行动（不能只是"主角决定做某事"）
- [ ] 中点是否改变了游戏规则（假胜利/假失败）
- [ ] 第二转折点是否是真正的最低点
- [ ] 高潮是否是主角主动的选择（不是巧合或外部救援）
- [ ] 复杂化是否递进升级（不是简单重复）

### 间隙设计检查（参考 gap-theory.md）
- [ ] 本卷使用了至少3种不同的间隙类型
- [ ] 间隙强度随章节递进（避免"间隙疲劳"）
- [ ] 不过度依赖反转（检查是否全是反转型间隙）
- [ ] 关键转折点都有明确的间隙设计
- [ ] 间隙与爽点配合（虐点铺垫→爽点释放）

### 戏剧结构检查
- [ ] 每章标注场景极性（正→负或负→正）
- [ ] 重要场景有节拍设计（行动→反应→Gap）
- [ ] 关键章节有抉择设计
- [ ] 价值转变与爽点对应

### 网文技法检查
- [ ] 每章有完整的主要内容描述
- [ ] 每章标注爽点类型和钩子
- [ ] 出场角色与characters.md一致
- [ ] 境界变化与outline.md一致
- [ ] 高潮节点数量达标（2-3个）
- [ ] 章节之间有关联承接
- [ ] 伏笔有埋有收

### 新实体规划检查（重要！）
- [ ] 每章明确规划新实体（新角色/新地点/新物品/新技能）
- [ ] 新角色来源于characters.md或明确标注为"龙套"
- [ ] 新地点来源于worldview.md或明确标注为"临时场景"
- [ ] 新物品/技能与世界观力量体系一致
- [ ] 无新实体的章节明确标注为空数组
```

## 使用说明

1. 用户说"细化第X卷"时，读取 `outline.md` 中该卷的信息，生成完整章节大纲
2. 每章必填：标题、主要内容、出场角色、爽点、钩子、境界、场景、字数、前后关联
3. 生成后更新 `outline.md` 中的"待细化卷"状态
4. chapter-writer 创作时读取本文件获取章节详细规划
